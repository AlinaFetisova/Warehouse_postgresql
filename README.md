# ?? Warehouse Management System (Django REST Framework)

Прогресивна бекенд-система (REST API) для автоматизації складського обліку та управління залишками товарів. Проєкт демонструє побудову безпечного API з розширеною фільтрацією, інтеграцією з реляційною базою даних та чітким розмежуванням прав доступу.

## ?? Основний функціонал
* **Гнучке управління складом:** Повне CRUD-управління (створення, читання, оновлення, видалення) товарами та категоріями через REST API.
* **Кастомна бізнес-логіка (educe_stock):** Реалізовано спеціальний ендпоінт для швидкого списання кількості товару на складі в реальному часі.
* **Розумний пошук та сортування:** Фільтрація товарів за категоріями, повнотекстовий пошук за назвою або артикулом (SKU) та динамічне сортування за ціною.
* **Рольова модель безпеки:** Автоматичне приховування та блокування інструментів редагування для неавторизованих користувачів.

## ??? Технологічний стек
* **Backend:** Python 3.x, Django REST Framework (DRF)
* **Database:** PostgreSQL (реляційна БД для забезпечення цілісності даних)
* **Tools:** Django Filters (для побудови динамічних пошукових запитів)

## ?? Архітектура та Ендпоінти (API Map)

| Метод | URL | Опис | Доступ |
| :--- | :--- | :--- | :--- |
| **GET** | \/api/products/\ | Отримати список усіх товарів з підтримкою фільтрації | Для всіх (Гість) |
| **POST** | \/api/products/\ | Створити новий товар на складі | Тільки Адмін |
| **POST** | \/api/products/{id}/reduce_stock/\ | Кастомне списання кількості товару | Тільки Адмін |
| **DELETE** | \/api/products/{id}/\ | Видалити товар з бази даних | Тільки Адмін |
| **GET** | \/api/categories/\ | Список доступних категорій | Для всіх (Гість) |

### ?? Приклади API-запитів:
* Пошук за назвою/SKU: \/api/products/?search=ноутбук\
* Сортування за ціною: \/api/products/?ordering=price\
* Фільтр за категорією: \/api/products/?category=1\

## ??? Архітектурні рішення (Software Engineering context)
* **Data Layer (PostgreSQL):** На відміну від файлових сховищ, використання повноцінної реляційної БД забезпечує ACID-транзакційність, сувору типізацію та безпеку зв'язків між товарами та категоріями (Foreign Keys).
* **Granular Access Control (Permissions):** Реалізовано архітектурний патерн \IsAuthenticatedOrReadOnly\. Логіка автентифікації повністю відокремлена від бізнес-логіки: на рівні налаштувань (\settings.py\) підключено \SessionAuthentication\ (для адмінів у браузері) та \BasicAuthentication\ (для сторонніх програм/Postman).
* **Business Logic Separation:** Кастомні операції (такі як \educe_stock\) інкапсульовані всередині методів ViewSet за допомогою декоратора \@action\. Це забезпечує чистоту коду та відповідність принципу Thin Controllers / Fat Models (або Fat ViewSets).
* **Data Integrity:** Валідація вхідних даних покладена на рівень серіалізаторів (Serializers), що гарантує захист бази даних від некоректних або шкідливих запитів.

## ?? Як запустити локально
1. Склонуйте репозиторій.
2. Налаштуйте та активуйте віртуальне оточення: \python -m venv venv\.
3. Встановіть залежності: \pip install -r requirements.txt\.
4. Переконайтеся, що локальний PostgreSQL запущено і створено БД \warehouse_db\.
5. Виконайте міграції: \python manage.py migrate\.
6. Створіть адміністратора: \python manage.py createsuperuser\.
7. Запустіть сервер: \python manage.py runserver\.

# 📦 Warehouse Management System (Flask)

Веб-додаток для автоматизації складського обліку. Проєкт демонструє роботу з backend-логікою на Python, інтеграцію з веб-інтерфейсом та обробку даних у реальному часі.

## 🚀 Основний функціонал

* **Інтелектуальне додавання**: при додаванні товару, який вже існує, система автоматично оновлює ціну та додає кількість до існуючого залишку.
* **Динамічне сортування**: перегляд товарів за назвою (алфавітом) або за ціною.
* **Гнучке управління**: можливість оновлювати кількість товару або видаляти позиції безпосередньо з інтерфейсу.
* **Пошук**: фільтрація товарів за назвою для швидкого знаходження позицій.
* **Експорт звітності**: генерація звіту у форматі `.txt` із розрахунком загальної вартості складу.

## 🛠 Технологічний стек

* **Backend**: Python 3.x, Flask
* **Frontend**: HTML5, CSS3, Jinja2
* **Storage**: JSON (серіалізація об'єктів для збереження даних)

## 📋 Архітектура
Проєкт побудовано на принципах **ООП**:
- Клас `Product`: модель даних товару.
- Клас `Warehouse`: логіка управління списком товарів та взаємодія з файловою системою.

## ⚙️ Як запустити
1. Склонуйте репозиторій.
2. Встановіть залежності: `pip install -r requirements.txt`.
3. Запустіть додаток: `python app.py`.
4. Відкрийте: `http://127.0.0.1:5000`.


## 🏗 Архітектурні рішення (Software Engineering context)

Проєкт реалізовано з чітким розділенням відповідальності (Separation of Concerns):

* **Data Layer (JSON)**: Використання файлової системи для персистентності даних без перевантаження коду складними БД на початковому етапі.
* **Business Logic (Models)**: Весь стан складу та операції з товарами інкапсульовані в класі `Warehouse`. Це дозволяє легко масштабувати логіку (наприклад, додати нові методи розрахунку податків чи знижок) без зміни веб-інтерфейсу.
* **Presentation Layer (Flask/Jinja2)**: Маршрутизація відокремлена від логіки обробки даних, що відповідає паттерну, близькому до **MVC** (Model-View-Controller).
* **Data Integrity**: Реалізовано перевірки на валідність вхідних даних (типи даних, від'ємні значення) для запобігання пошкодженню JSON-сховища.

## 🛠 Backend Engineering Highlights

* **RESTful Routing**: Використання HTTP методів (GET, POST) для керування ресурсами.
* **Error Handling**: Базова обробка виключень при роботі з файлами та некоректним введенням користувача.
* **State Management**: Синхронізація об'єктів у пам'яті з файлом на диску після кожної значущої транзакції (додавання, видалення, оновлення).
